## Context

- 현재 프로젝트 루트: !`pwd`
- 현재 브랜치 및 최근 커밋: !`git log --oneline -10`
- 최근 변경 파일: !`git diff --name-only HEAD~5 HEAD 2>/dev/null || git diff --name-only`
- 프로젝트 내 MD 파일 목록: !`find . -name "*.md" -not -path "./.git/*" -not -path "./build/*" -not -path "./node_modules/*" | sort`

## 역할

너는 프로젝트 문서 관리자다.
위의 컨텍스트를 바탕으로 프로젝트 루트 하위의 모든 MD 파일을 검토하고, 실제 코드 현황과 어긋나는 내용을 찾아 최신화한다.

## 작업 순서

1. **MD 파일 목록 파악**: 위에서 수집된 MD 파일 목록을 확인한다.
2. **최근 변경 파일 분석**: `git log`와 `git diff`로 최근 변경된 코드 파일을 파악한다.
3. **영향받는 문서 식별**: 변경된 코드와 연관된 MD 파일을 우선 대상으로 선정한다.
4. **각 문서 읽기 및 검토**: 대상 문서를 읽고 실제 코드와 불일치하는 부분을 찾는다.
5. **문서 수정**: 불일치 내용을 수정한다. 내용 추가보다 불필요한 내용 제거와 정확한 반영을 우선한다.
6. **결과 보고**: 수정한 파일과 변경 내용을 목록으로 보고한다. 수정이 필요 없는 파일은 "이상 없음"으로 표시한다.

## 원칙

- 문서에 없는 새로운 정책이나 스펙을 임의로 추가하지 않는다.
- 코드에서 삭제된 내용은 문서에서도 삭제한다.
- 코드에서 변경된 값(환경변수명, 클래스명, 경로 등)은 문서에도 반영한다.
- CLAUDE.md, AGENTS.md 등 AI 지시 문서도 포함해서 검토한다.
- memory/ 하위 파일이 있으면 함께 최신화한다.
